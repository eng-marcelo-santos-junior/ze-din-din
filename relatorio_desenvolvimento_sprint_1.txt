================================================================================

Criada a fundação completa da aplicação Flask com arquitetura modular,
infraestrutura Docker e página inicial pública. A aplicação sobe com um
comando, conecta no PostgreSQL e serve a landing page do Zé Din Din via
Blueprint "main".

3. COMO TESTAR MANUALMENTE
================================================================================

1. Subir a aplicação:

   make build
   make up

2. Verificar se os containers estão rodando:

   docker compose ps

   Esperado: "web" e "db" com status "running".

3. Verificar logs:

   make logs

   Esperado: Running on http://0.0.0.0:5000

4. Acessar no navegador:

   http://localhost:5000

   Esperado: landing page do Zé Din Din com hero verde,
   6 cards de features e seção de 3 passos.

5. Testar conexão com o banco:

   make psql

   Dentro do psql:
     \l    (lista os bancos)
     \q    (sair)

   Esperado: bancos "zedindin_dev" e "zedindin_test" presentes.


4. COMANDOS NECESSÁRIOS
================================================================================

# Primeira vez
cp .env.example .env
make build
make up

# Dia a dia
make up          # sobe containers
make down        # para containers
make logs        # acompanha logs do Flask

# Banco
make psql        # acessa o PostgreSQL diretamente

# Migrations (a partir do Sprint 2)
make migrate-init      # flask db init (apenas uma vez)
make migrate           # flask db upgrade
make migration msg=""  # flask db migrate -m "descricao"

# Testes (a partir do Sprint 2)
make test
make test-cov

# Qualidade de código (requer requirements-dev.txt)
make lint
make format
